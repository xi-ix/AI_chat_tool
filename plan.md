# 桌面问答工具计划

## 目标
做一个常驻后台的桌面工具，通过全局快捷键快速唤出问答窗口，输入问题后调用 API 返回答案。

## 优先级
1. 如果双端支持的成本可控，优先直接做 macOS + Windows 双端。
2. 如果双端推进中遇到平台差异或打包成本较高，先完成 macOS 版本，再补 Windows。

## 推荐技术路线
- 桌面框架：Tauri
- 前端：轻量 Web UI
- 通信方式：前端调用 Tauri 命令，后端负责 API 请求和系统能力
- API：兼容 OpenAI 的 HTTP 接口

## 第一阶段：Mac 可用原型
1. 搭建 Tauri 项目骨架。
2. 做一个简洁的问答窗口，支持输入问题、提交、显示结果。
3. 注册全局快捷键，按下后唤出窗口。
4. 启动后默认隐藏在后台，显示托盘或菜单栏入口。
5. 接入 API 配置和请求流程。
6. 完成 macOS 开机自启。

## 第二阶段：双端扩展
1. 抽离平台相关能力，统一快捷键、启动、窗口控制接口。
2. 补 Windows 开机自启。
3. 验证 Windows 下的快捷键、窗口唤出和后台常驻行为。
4. 处理两端打包和安装差异。

## 需要先确认的细节
1. 默认全局快捷键用什么组合。：设置界面里可修改。
2. API Key 用系统钥匙串还是本地配置文件保存。：本地配置文件。
3. 是否需要托盘菜单里的设置入口。：设置界面里可修改。
4. 是否需要保存历史问题和回答。：保存到本地数据库。

## 当前结论
先按 Mac 原型推进，同时把架构留出双端扩展能力；如果实现和打包成本可控，再同步支持 Windows。
